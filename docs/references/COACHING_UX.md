# Reference: coaching UX copy & algorithms

## Tone

- **Supportive, direct** — Short sentences; celebrate consistency and honest effort.
- **Transparent** — Label algorithmic rows: “Suggested (from your last 3 sessions)”.

## Suggestion disclosure

- Always show: **basis** (last session date, n sessions), **assumptions** (equipment increment).
- Allow **dismiss** and **“never suggest for this exercise”** (stored in preferences).

## Inputs that improve suggestions (prioritize in UI)

1. **RPE or RIR** after hard sets (optional field).
2. **Equipment step** (gym has 2.5 lb plates vs not).
3. **Failure vs technical stop** (optional note).

## Progressive overload UI affordances

- **Side-by-side**: last session vs this session (same exercise).
- **Trend sparkline** or simple table: volume or top set per week.
- **Next session card** on home: “Bench press — try 3×8 @ 140” with tap to start workout.

## Accessibility

- Targets and buttons large enough for sweaty hands / mobile; respect `prefers-reduced-motion`.
